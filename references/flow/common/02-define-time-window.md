# 公共SOP 2：明确时间窗口

> 确定分析的数据时间范围和时效性

---

## 目的

确保使用正确的数据，避免用错"昨日"/"今日"/"最近交易日"。

---

## 操作步骤

### Step 1：确定分析日期

```
【分析日期】= 用户指定日期 或 当前日期
```

### Step 2：计算最近交易日

最近交易日 = 当前日期前最后一个A股交易日

**规则：**
- 周一~周五（非节假日）= 交易日
- 周六、周日 = 非交易日 → 往前找周五
- 法定节假日 = 非交易日 → 往前找最后一个交易日

**示例：**

| 当前日期 | 最近交易日 |
|----------|------------|
| 周一 | 今天 |
| 周日 | 上周五 |
| 春节后第一个交易日 | 节后当天 |
| 国庆后第一个交易日 | 节后当天 |

### Step 3：确定时段

| 时段 | 时间 | 操作建议 |
|------|------|----------|
| 盘前 | 08:00前 | 用最近交易日数据 |
| 竞价 | 09:15-09:25 | 观察竞价信号 |
| 盘中 | 09:30-11:30 / 13:00-15:00 | 谨慎追涨 |
| 盘后 | 15:00后 | 用今日收盘数据 |
| 非交易日 | 周末/节假日 | 用最近交易日数据 |

### Step 4：T+1计算（如适用）

```
买入日(T日) → 可卖出日(T+1日)

注意：
- T日买的不能T日卖
- 周五买 → 下周一才能卖
- 节假日后第一个交易日买 → 下一个交易日才能卖
```

---

## 日期标注

所有输出必须包含：

```markdown
【分析日期】YYYY-MM-DD
【最近交易日】YYYY-MM-DD
【时段】盘前/盘中/盘后/非交易日
【T+1说明】（如涉及持仓）
```

---

## 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| 周日说"昨日" | 以为是周六 | 明确是最近交易日 |
| 混淆节前节后 | 用错数据 | 明确交易日边界 |
| T+1算错 | 无法成交 | 下一交易日才可卖 |
