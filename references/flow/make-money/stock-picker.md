# 选股

> 筛选有机会的股票，给出理由和预期收益

## 触发词

- "有什么能赚钱"
- "推荐股票"
- "有什么好股"
- "帮我选股"
- "高PR选股"

## 前置确认

- [ ] 确认持仓周期（短线/中线/长线）
- [ ] 确认风险偏好（保守/平衡/激进）
- [ ] 确认仓位上限

## 分析步骤

### Step 1：三维看市场

判断大盘环境是否适合选股：

| 信号 | 环境 | 操作 |
|------|------|------|
| 🟢🟢🟢 | 强势 | 积极选股 |
| 🟢🟡🟢 | 震荡偏强 | 精选个股 |
| 🟡🟡🟡 | 震荡 | 控制仓位 |
| 🔴🔴🔴 | 弱势 | 休息/空仓 |

### Step 1.5：先分清是趋势票、性价比票，还是修复观察票

短中线选股时，先不要急着比 PE 高低，先做分类：

- **趋势票**
  主线题材清晰，趋势向上，回踩后仍有承接。
- **性价比票**
  估值更舒服，但需要新的催化或板块轮动来点火。
- **修复观察票**
  逻辑存在，但图形未修复、上方套牢盘重，不排在前面。

默认优先级：

`趋势票 > 性价比票 > 修复观察票`

### Step 2：三重过滤器筛选

**过滤器1：基本面+趋势+流动性**

使用 `mx-stocks-screener` 筛选：

```bash
python3 get_data.py \
  --query "股价在20日均线上方，成交额排名前50，市盈率<40的A股" \
  --select-type A股
```

> 高换手率确保趋势有资金支撑

补充规则：

- 当主线资金明显偏向电子、算力、半导体材料等高景气方向时，低估值但不在主线里的制造票默认降级到观察池。
- 低 PE / 低 PEG 只能在候选股都通过趋势与催化筛选后再作为排序微调项，不能直接翻盘。

**过滤器2：技术数据提取**

使用 `mx-finance-data` 提取技术位：

```bash
python3 get_data.py "查询 [股票代码] 最新股价、20日均线位置、BOLL轨道"
```

**过滤器3：情报合成**

使用 `mx-financial-assistant` 结合技术+催化剂：

```bash
python3 generate_answer.py \
  --query "分析 [个股] 高盈亏比机会。结合：1. [近期催化剂]；2. 业绩；3. 均线支撑。给出买入/目标/止损建议。" \
  --deep-think
```

### Step 3：四维评级

对候选股进行四维评级：

```
综合分 = 机会⭐×30% + 安全⭐×25% + 确定⭐×25% + 舒适⭐×20%

4.5+ → ⭐⭐⭐⭐⭐ 立即买
3.5+ → ⭐⭐⭐⭐ 可以买
2.5+ → ⭐⭐⭐ 观望
```

### Step 4：R3红线过滤

```
安全⭐ < 3 → 不买
安全⭐ ≥ 3 + 机会⭐ ≥ 3 + 确定⭐ ≥ 3 → 可以买
```

## 选股标准速查

| 维度 | 标准 |
|------|------|
| 基本面 | PE<25（或行业平均），Q1/年增长>20% |
| 技术面 | 价格在20/60日均线的2-3%范围内 |
| 催化剂 | 明确上涨理由（政策/断供/同行带动） |
| 风险收益 | 目标收益 ≥ 3×止损距离 |

## 硬规则检查

- ❌ 安全⭐<3 → 不推荐
- ❌ 成交额<5000万 → 不推荐
- ❌ 弱势大盘 → 休息
